# Release

This directory holds release-specific planning notes and the release playbook.

## Purpose

Use these docs when you need to turn a release task list into a tagged release or a published note.

Release-specific planning stays here.
Project-wide backlog work stays in `docs/roadmap/IMPLEMENTATION_ROADMAP.md`.

## Current Files

- `RELEASE_PLAYBOOK.md`: release workflow and checklist
- `releases/README.md`: index of version-specific release notes
- `releases/v1.2.6.md`: current release task list

## Reading Order

1. `RELEASE_PLAYBOOK.md`
2. `releases/README.md`
3. the version note you are working on
